## 1. 全体コンセプト

### 新しいLifeVillageの定義

> 公務員のお金の悩みを、まるごと解決する無料相談窓口

生命保険販売に特化した事業から、**公務員向け総合マネー相談窓口**へとポジションを刷新する。  
公務員特化という軸は維持しつつ、扱う領域を4つに拡張する。

### 扱うサービス領域

| 領域 | 内容 |
|------|------|
| 保険 | 生命保険・医療保険・がん保険・損害保険 |
| 資産形成 | つみたてNISA・iDeCo（公務員は確定拠出年金）・投資信託 |
| 公務員特有の制度 | 共済年金・退職金の活用・公務員向けローン・住宅購入 |
| ライフプランニング | 家計診断・ライフイベント別マネープラン（育児・マイホームなど） |

### 収益モデル

- **無料相談で集客**し、以下で収益化
  - 保険販売手数料（生命保険・医療保険等）
  - 金融商品アフィリエイト（NISA口座開設・ローン・投資信託紹介等）

### LPと診断アプリの関係

```
LP（集客・コンセプト訴求）
  └── 診断CTA（メイン導線）
        └── 診断アプリ（公務員向け総合マネー診断）
              └── 診断結果 → 無料相談予約（CV）
```

LP骨子を先に確定し、診断の質問・結果タイプがLPのメッセージと整合するよう並行実装する。

## 3. 診断アプリ設計（リニューアル）

### コンセプト

「あなたに今必要なお金の相談」を5問で診断し、優先度の高い領域と次のアクション（相談予約）を提示する。

### 診断フロー

```
スタート画面
  → Q1: 職種（教員 / 地方公務員 / 国家公務員 / その他）
  → Q2: 年代（20代 / 30代 / 40代 / 50代）
  → Q3: 家族構成（独身 / 既婚・子なし / 既婚・子あり）
  → Q4: 今気になっていること（複数選択）
       ・保険の見直し / 老後資金 / 資産運用 / 住宅購入
       ・退職金・共済の活用 / 家計の見直し / 子どもの教育費
  → Q5: 現在の取り組み状況（何もしていない / 少し始めた / 見直したい）
  → 診断結果画面
```

### 診断結果タイプ

| タイプ | 概要 | 主な提案内容 |
|-------|------|-------------|
| 保険見直し優先 | 今の保障が公務員制度と重複している可能性あり | 共済との重複確認・保険の最適化 |
| 資産形成スタート | iDeCo・NISAで効率的な積立を提案 | 積立額シミュレーション・口座開設案内 |
| 退職金・共済活用 | 退職後の収支シミュレーションが必要 | 退職金運用・年金受取プラン |
| ライフプラン再設計 | 住宅購入・教育費を含めた総合プランが必要 | キャッシュフロー表作成・FP相談 |

### 結果画面のCTA

「この診断結果をもとに無料相談する」ボタンをメインCTAとして配置。診断結果のサマリー（タイプ名）をURLパラメータで相談予約フォームに引き継ぐ。

### 実装対象ファイル（Next.js app）

- `app/diagnosis/page.tsx` — スタート画面（リニューアル）
- `app/diagnosis/quiz/page.tsx` — 質問画面（5問に再設計）
- `app/diagnosis/result/page.tsx` — 結果画面（4タイプに対応）
- `lib/` — 診断ロジック・タイプ判定

## 4. 実装方針

### 進め方（同時並行型）

1. LP骨子（HTML構成・コピー）を確定する
2. LP骨子をガイドラインに診断アプリの質問・結果タイプを実装する
3. LP v2にスタイルを当てて完成させる
4. LP上の診断CTAを実装済みの診断アプリに接続する

### 既存ファイルの扱い

| ファイル | 対応 |
|---------|------|
| `lp/v2/index.html` | 全セクション実装（現在は骨格のみ） |
| `app/diagnosis/quiz/page.tsx` | 質問を生命保険診断から総合マネー診断に再設計 |
| `app/diagnosis/result/page.tsx` | 結果タイプを4タイプに再設計 |
| `TechVillage/businesses/life_village.md` | 事業概要を更新 |
